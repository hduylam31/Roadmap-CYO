03 — RISKS
Rủi ro & Giải pháp
Những rủi ro có thể khiến dự án chệch hướng — và cách phòng tránh.
Critical
Token Cost bùng nổ khi scale users
SRD estimate Claude API ~$14/mo dựa trên giả định 5-20 users, batch processing weekly. Nhưng nếu 15 Marketing users dùng SegmentChat (on-demand, real-time LLM) hàng ngày — mỗi chat ~$0.01, mỗi người 20 queries/ngày → $0.01 × 20 × 15 × 22 ngày = $66/mo chỉ riêng SegmentChat. Cộng InsightGen (Opus, đắt nhất) + ContentGen → token cost có thể tăng 5-10x so với estimate.
❌ Hiện tại — Full LLM
  • Mọi query đều gọi Claude API
  • "Khách VIP HN" → Sonnet parse → ES query
  • Mỗi lần = ~2K-5K tokens
  • Scale 15 users = chi phí tuyến tính
  • $14/mo → có thể $70-140/mo
✓ Giải pháp — Text-to-SQL/ES
  • Query đơn giản → parse trực tiếp, không gọi LLM
  • "Khách VIP HN" → Rule-based → ES query
  • Zero token cost cho 80% queries
  • Scale vô hạn users, chi phí = $0
  • LLM chỉ cho queries phức tạp (20%)
→ Giải pháp đề xuất: Hybrid Text-to-ES
Xây Query Classifier trước khi gọi LLM. Phân loại query thành 2 tier:

Tier 1 — Rule-based (target: 80% queries, zero LLM cost):
Các pattern phổ biến có thể parse bằng regex/keyword matching:
• Filter đơn: "khách VIP", "khách HN", "nữ 25-35" → map trực tiếp sang ES query DSL
• RFM segments: "VIP", "Champion", "At-risk", "Lost" → lookup bảng segment có sẵn
• Saved segments: "segment tháng trước", "top spenders" → query SQLite
• Aggregations: "bao nhiêu khách HCM", "tổng revenue Q3" → template ES agg query

Tier 2 — LLM (target: 20% queries, controlled cost):
Chỉ dùng Claude khi query thực sự phức tạp:
• Multi-condition phức tạp: "khách mua polo > 2 lần ở HN nhưng chưa mua Q2, follow Zalo"
• Ambiguous intent: "khách có vẻ sắp churn" (cần LLM reason)
• Iterative refinement: "thu hẹp thêm theo AOV" (cần context conversation)

Critical
BQ Access chậm → toàn bộ timeline trượt
Critical path: BQ access → dbt models → ES indexing → AI Agents → Demo. Nếu BI team (Anh Bình) không cấp BQ access trong tuần 1 → mọi thứ shift. Demo 08/05 chỉ cách 3.5 tuần — không có buffer.
→ Mitigation
PO push BQ access ngày 15/04 (ngày mai). Nếu không response trong 48h → escalate CTO. Fallback: dùng sample CSV export từ BQ để unblock dbt development song song.
Critical
AI Hallucination → Marketing ra quyết định sai
InsightGen (Opus) có thể bịa metrics: "doanh thu Polo tăng 50%" trong khi thực tế chỉ 12%. Nếu PO approve mà không catch → Marketing chạy campaign sai → thiệt hại brand + budget.
→ Mitigation đã có trong SRD
QualityCheck (Haiku) fact-check mọi số liệu trước khi đến PO. Approval Workflow (#8) bắt buộc human review. PO cần cross-check ≥ 3 metrics/insight với BQ raw data trong tháng đầu để calibrate trust level.
Medium
Cowork MCP chưa verify SSE + OAuth
SRD ghi rõ: "cần verify Cowork support SSE transport + OAuth 2.0". Đây vẫn là open question. Nếu Cowork không support → Demo v0.15 phải chuyển sang REST API + CLI.
→ Mitigation
Test Cowork MCP connection trong tuần 1 (trước khi invest 6-8 ngày vào MCP Server). Fallback: Demo qua REST API + Postman/CLI — ít ấn tượng hơn nhưng functional.
Medium
PII leak qua Claude API → vi phạm PDPA
CYO xử lý 6M+ PII records. Claude API servers ở US. Nếu code bug gửi phone/name lên Claude thay vì c360_id → vi phạm Nghị định 13 về chuyển dữ liệu xuyên biên giới. Giải pháp Text-to-SQL/ES (Risk #1) cũng giúp giảm rủi ro này — ít query qua LLM = ít cơ hội leak PII.
→ Mitigation
PII Response Filter strip phone/name/email trước khi gửi Claude. Whitelist approach — chỉ các fields được phép mới pass qua. ES read-only user cho agents — không thể modify data. PO cần verify PII filter trước Demo bằng test case cụ thể.
04 — GLOSSARY
Thuật ngữ quan trọng
Giải nghĩa ngắn gọn, có ví dụ cụ thể trong dự án CYO.
Kiến trúc & Hạ tầng
MCP Server
Model Context Protocol
Giao thức cho phép AI (Claude) kết nối và dùng công cụ bên ngoài. CYO tạo MCP Server để Cowork truy vấn data, tạo segment, xem insight qua chat.
PO gõ "cho tôi xem insight tuần này" → Claude gọi MCP tool → trả kết quả.
Cowork
Claude Cowork (Anthropic)
App desktop chat với Claude, kết nối hệ thống ngoài qua MCP. Kênh chính cho power users (CTO, PO, Marketing Lead).
CTO hỏi "tại sao segment này mua Polo nhiều Q3?" → Claude truy vấn ES + Graphiti.
Keycloak
Open-source Identity Provider
Hệ thống đăng nhập & phân quyền. YODY đã có sẵn. CYO tạo realm/client quản lý 5 roles (Admin, PO, Lead, Executor, Viewer).
PO đăng nhập → Keycloak check role → cho approve insight nhưng không manage users.
VPC / WAF / ALB
AWS Network Stack
VPC = mạng riêng cách ly. WAF = tường lửa web. ALB = bộ phân tải. CYO chỉ cho IP văn phòng YODY truy cập.
Data Pipeline & Xử lý
dbt
data build tool
Công cụ biến đổi data bằng SQL. Tổ chức: staging → intermediate → mart. "Nhà máy chế biến" data thô thành data sạch.
Raw customers từ BQ → clean, dedup, tính RFM → output mart_customer_360 (6M+ rows).
Dagster
Orchestrator
Công cụ điều phối — tự động chạy các bước xử lý data theo lịch. Quản lý: chạy dbt, kéo API data, đẩy vào ES.
Dagster chạy 3×/tuần: Gọi Meta API → chạy dbt → đẩy vào Elasticsearch.
Elasticsearch (ES)
Search & Analytics Engine
Tìm kiếm & phân tích data cực nhanh. CYO lưu Customer 360 + ad signals để AI agents truy vấn.
"Tìm khách HCM, nữ, 25-35, mua >3 lần" → ES trả kết quả < 2 giây từ 6M records.
Graphiti + Neo4j
Knowledge Graph
Neo4j = database đồ thị lưu quan hệ. Graphiti = thư viện giúp AI xây "bộ nhớ" từ data. Lưu: Segment ↔ Product ↔ Channel.
"Segment VIP_HN thích Polo Classic, mua nhiều qua Zalo, peak Q3" → AI dùng kiến thức này gợi ý content.
PII / PII Masking
Personally Identifiable Information
Thông tin định danh: tên, SĐT, email. PII Masking = che giấu PII trong logs/cloud. CYO KHÔNG BAO GIỜ gửi PII lên Claude API.
Log ghi "customer_id=CUS_00123" thay vì "Nguyễn Văn A, 0912xxx".
RFM
Recency – Frequency – Monetary
Mô hình phân loại KH: mua lần cuối bao lâu, mua bao nhiêu lần, chi bao nhiêu. Mỗi yếu tố 1-5 điểm.
RFM 555 = "VIP Champion". RFM 111 = "Lost / Hibernating".
AI Agents
Agent (AI Agent)
—
Chương trình AI chuyên biệt 1 nhiệm vụ. CYO chia nhỏ thay vì 1 AI làm tất cả (dễ sai).
DataAnalyst (truy vấn) → InsightGen (phân tích) → QualityCheck (kiểm tra) → ContentGen (viết).
Hallucination
—
AI "bịa" thông tin không có trong data. Lý do CYO có QualityCheck agent + Approval workflow.
InsightGen nói "doanh thu tăng 50%" → QualityCheck check lại → thực tế chỉ 12% → flag sai.
Sonnet / Opus / Haiku
Claude Model Tiers
Haiku = nhanh, rẻ (check nhanh). Sonnet = cân bằng (truy vấn, viết). Opus = mạnh nhất, đắt nhất (phân tích sâu).
DataAnalyst = Sonnet. InsightGen = Opus. QualityCheck = Haiku.
Product & Quy trình
UAT
User Acceptance Testing
Marketing team dùng thử CYO và xác nhận "OK dùng được" hoặc "cần sửa". Bước cuối trước go-live.
RBAC
Role-Based Access Control
Phân quyền theo vai trò. 5 roles: Admin, PO, Marketing Lead, Executor, Viewer — mỗi role quyền khác nhau.
DPIA
Data Protection Impact Assessment
Đánh giá tác động bảo vệ dữ liệu — bắt buộc theo Nghị định 13 khi xử lý data cá nhân quy mô lớn (6M+ customers).
Acceptance Criteria
AC
Tiêu chí chấp nhận. Format: GIVEN (điều kiện) → WHEN (hành động) → THEN (kết quả).
GIVEN PO xem insight → WHEN nhấn "Approve" → THEN status = approved + timestamp ghi lại.
Critical Path
—
Chuỗi công việc phụ thuộc nhau — bất kỳ cái nào trễ = toàn bộ dự án trễ.
BQ access → dbt → ES → Agents → Demo. BQ chậm 3 ngày = Demo chậm 3 ngày.
P0 / P1 / P2
Priority Levels
P0 = Critical, fix ngay. P1 = Important, fix trong sprint. P2 = Nice-to-have, backlog.
P0: "Pipeline fail" → fix ngay. P1: "Font sai" → sprint. P2: "Dark mode" → backlog.
