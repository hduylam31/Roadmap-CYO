00 — BỐI CẢNH
Tại sao cần CYO?
Hiện trạng dữ liệu hỗn độn → Cần hệ thống tập trung để khai thác 6M+ customer records bằng AI.
❌ Thực trạng — Vấn đề
  • Chưa có CRM thực thụ. Dữ liệu KH nằm rải rác trong hệ thống cũ Unicorn — vốn không được thiết kế cho quản trị khách hàng.
  • Các kênh tương tác (Facebook, Zalo, SMS...) bị rời rạc, dữ liệu chia cắt, không thể khai thác cross-channel.
  • Không có Single Source of Truth — Marketing tự đào bới data thủ công, chậm và thiếu chính xác.
  • Con người không thể xử lý hiệu quả 6M+ records — cần AI để làm thay.
  • Nguy cơ vi phạm PDPA / Nghị định 13 nếu không có hệ thống PII masking rõ ràng.
✓ Giải pháp — CYO = "See You" (Thấu hiểu khách hàng)
  • Tập trung toàn bộ điểm chạm KH về 1 mối duy nhất → Customer 360° profile.
  • AI phân tích 6M+ records tự động → Insight thay thế & tối ưu năng lực con người.
  • Kiến trúc Multi-Agent: DataAnalyst → InsightGen → QualityCheck → ContentGen.
  • Triển khai MCP-first: Leaders chat trực tiếp với dữ liệu qua Claude/Cowork.
  • PII masking bắt buộc: Chỉ Customer ID được gửi lên Cloud AI — tuân thủ tuyệt đối Nghị định 13.

08 — RISKS · SUGGESTIONS
Rủi ro · Đề xuất cải tiến
2 nhóm: rủi ro kỹ thuật/kinh doanh cần theo dõi với mitigation plan, và đề xuất chủ động cho Phase 2.
Critical
Token cost vượt budget khi scale
100% LLM queries = chi phí tuyến tính. Scale 15 users → $70–140/tháng chỉ riêng Claude API. InsightGen (Opus) đắt nhất — 1 insight batch ≈ $0.80.
→ Mitigation
Build Query Classifier — Phase 2 P1 (Q3). MVP chấp nhận 100% LLM với 3–5 users (cost ~$14–20/mo). Trước khi mở 15 users → Hybrid 80/20 phải ready. Budget cap: $300/mo total.
Critical
BQ Access chậm → toàn bộ timeline trượt
Critical path cứng: BQ access (BI Team T1) → dbt mart tables → ES sync → AI Agents → Demo 08/05. T3 = 0 ngày nghỉ lễ đã xóa hết buffer. Delay 1 ngày ở T1 = Demo trễ 1 ngày.
→ Mitigation
Escalate ngay 17/04 nếu BQ chưa sẵn. Fallback: staging BQ dataset (synthetic 50K records) để unblock Developer Team build pipeline. PO track BQ access như P0 task.
Critical
AI Hallucination → Marketing ra quyết định sai
InsightGen (Opus) có thể bịa số liệu plausible nhưng sai. Approver không đủ context để phát hiện → Marketing chạy campaign sai → thiệt hại brand + budget.
→ Mitigation
QualityCheck Agent (Haiku) fact-check mọi số liệu với ES. Approval Workflow bắt buộc. Mỗi insight có Evidence section + confidence score. PO cross-check ≥ 3 metrics/insight trong tháng đầu.
Medium
Cowork MCP chưa verify SSE + OAuth 2.0
SRD ghi "cần verify Cowork support SSE transport + OAuth 2.0" — vẫn là open question tại thời điểm viết roadmap. Không support → toàn bộ MCP layer phải chuyển sang REST API.
→ Mitigation
Test Cowork MCP connection T5 (11–15/05) — ngay sau Demo, trước khi invest build thêm tools. Fallback: Demo qua REST API + Postman/CLI — functional nhưng kém UX hơn.
Medium
PII leak qua Claude API → vi phạm Nghị định 13
CYO xử lý 6M+ PII records. Claude API servers đặt tại US. Một bug nhỏ gửi phone/name thay vì c360_id → vi phạm cross-border data transfer theo Nghị định 13/2023.
→ Mitigation
PII Response Filter whitelist approach — chỉ approved fields mới pass. Developer verify filter với test case cụ thể trước Demo. Legal review T5–T6. DPIA sign-off ≤ 19/06 (T10).
Medium
Graphiti chưa có production POC ở scale 6M+ nodes
Graphiti là thư viện Knowledge Graph còn mới. Không có public benchmarks ở scale 6M customer nodes + relationship edges. Nếu latency >5s → Knowledge Graph layer phải redesign toàn bộ.
→ Mitigation
Chạy POC load test với 100K nodes trước T4. Nếu latency >5s → fallback về ES-only (vector search, bỏ Knowledge Graph). Quyết định trước Demo 08/05 để không block go-live.
Medium
Marketing adoption thấp sau go-live
5 users active là target tối thiểu. Nếu Marketing team không hình thành habit dùng CYO trong 4 tuần đầu → dự án thành công kỹ thuật nhưng thất bại về business value. Adoption mà không đo được = không biết cải tiến gì.
→ Mitigation
PO embed CYO vào weekly marketing meeting — mỗi họp đầu tuần dùng 1 insight từ CYO để kickoff. Training T9 phải hands-on, không phải slideshow. Nếu <5 users active sau tuần 2 → PO interview ngay, không chờ tổng kết tháng.

09 — GLOSSARY
Thuật ngữ quan trọng
Giải nghĩa ngắn gọn, có ví dụ cụ thể trong dự án CYO.
Kiến trúc & Hạ tầng
MCP Server
Model Context Protocol
Giao thức cho phép AI (Claude) kết nối và dùng công cụ bên ngoài. CYO tạo MCP Server để Cowork truy vấn data, tạo segment, xem insight qua chat.
Team Product gõ "cho tôi xem insight tuần này" → Claude gọi MCP tool → trả kết quả.
Cowork
Claude Cowork (Anthropic)
App desktop chat với Claude, kết nối hệ thống ngoài qua MCP. Kênh chính cho power users (CTO, Team Product, Marketing Lead).
CTO hỏi "tại sao segment này mua Polo nhiều Q3?" → Claude truy vấn ES + Graphiti.
Keycloak
Open-source Identity Provider
Hệ thống đăng nhập & phân quyền. YODY đã có sẵn. CYO tạo realm/client quản lý 5 roles (Admin, PO, Lead, Executor, Viewer).
Data Pipeline & Xử lý
dbt
data build tool
Công cụ biến đổi data bằng SQL. staging → intermediate → mart. "Nhà máy chế biến" data thô thành data sạch.
Raw customers từ BQ → clean, dedup, tính RFM → output mart_customer_360 (6M+ rows).
Dagster
Orchestrator
Công cụ điều phối — tự động chạy các bước xử lý data theo lịch.
Dagster chạy 3×/tuần: Gọi Meta API → chạy dbt → đẩy vào Elasticsearch.
Elasticsearch (ES)
Search & Analytics Engine
Tìm kiếm & phân tích data cực nhanh. CYO lưu Customer 360 + ad signals để AI agents truy vấn.
"Tìm khách HCM, nữ, 25-35, mua >3 lần" → ES trả <2 giây từ 6M records.
PII / PII Masking
Personally Identifiable Information
Thông tin định danh: tên, SĐT, email. PII Masking = che giấu. CYO KHÔNG BAO GIỜ gửi PII lên Claude API. Tuân thủ Nghị định 13/2023/NĐ-CP.
Log ghi "customer_id=CUS_00123" thay vì "Nguyễn Văn A, 0912xxx".
RFM
Recency – Frequency – Monetary
Mô hình phân loại KH: mua lần cuối bao lâu, mua bao nhiêu lần, chi bao nhiêu. Mỗi yếu tố 1-5 điểm. BI Team owns formula + validation.
RFM 555 = "VIP Champion". RFM 111 = "Lost / Hibernating".
Graphiti
Knowledge Graph Library
Thư viện xây dựng Knowledge Graph cho AI. CYO dùng để lưu quan hệ giữa customers, products, channels. Cần POC scale test trước T4.
Graphiti biết "KH mua áo Polo thường mua quần Kaki cùng lúc" → insight cross-sell.
AI Agents
Agent (AI Agent)
—
Chương trình AI chuyên biệt 1 nhiệm vụ. CYO chia nhỏ thay vì 1 AI làm tất cả (dễ sai).
DataAnalyst (truy vấn) → InsightGen (phân tích) → QualityCheck (kiểm tra) → ContentGen (viết).
Sonnet / Opus / Haiku
Claude Model Tiers
Haiku = nhanh, rẻ (check nhanh). Sonnet = cân bằng (truy vấn, viết). Opus = mạnh nhất, đắt nhất (phân tích sâu).
DataAnalyst = Sonnet. InsightGen = Opus. QualityCheck = Haiku.
Product & Quy trình
UAT
User Acceptance Testing
Marketing team dùng thử CYO và xác nhận "OK dùng được". Bước cuối trước go-live. Team Product observe trực tiếp để catch UX issues.
RACI
Responsible, Accountable, Consulted, Informed
Ma trận trách nhiệm. R = làm, A = chịu trách nhiệm, C = tham khảo, I = được thông báo.
DPIA
Data Protection Impact Assessment
Đánh giá tác động bảo vệ dữ liệu — bắt buộc theo Nghị định 13 khi xử lý PII quy mô lớn. Legal Team sign-off chậm nhất T10 (19/06).

BQ access (T1) → dbt → ES → Agents → Demo (08/05). BQ chậm 3 ngày = Demo trễ 3 ngày.

4 Must Pass + 2 Should Pass. Thiếu bất kỳ Must Pass → No-Go tự động.
